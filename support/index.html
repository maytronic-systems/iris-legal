Support

IRIS is a visualization and monitoring dashboard. Some Android data sources require explicit user-granted access (e.g., Usage Access). This page explains the most common issues.

Contact: SUPPORT@YOURDOMAIN.TLD — include device model, Android version, and IRIS app version.

FAQ

Why does IRIS ask for “Usage Access”?

Certain dashboards rely on Android usage statistics APIs. Android requires explicit user permission (“Usage Access”) to read those aggregated usage signals.

IRIS uses this access to visualize usage and related summaries; it does not claim to control other apps.

Network traffic values look different from my carrier / router.

Android-provided network statistics may differ from external measurements (carrier, router, VPN, etc.) due to sampling windows, rounding, or measurement boundaries.

Why do some apps show limited data?

Some system constraints and OEM implementations limit visibility for certain apps or time windows. If you recently installed IRIS, it may also take time to build meaningful historical summaries.

Does IRIS block trackers or prevent network connections?

No. IRIS focuses on visualization and analysis. It provides dashboards and summaries so you can make your own decisions.

I purchased Pro but features are locked.

Ensure you are signed in with the same Google account used for purchase. Then try restoring purchases (if available) and restart the app. If it still fails, contact support with your order ID (optional).

Troubleshooting checklist

  1. Update IRIS to the latest version.
  2. Restart the device (ensures system stats providers are refreshed).
  3. Grant required access (Usage Access, notifications for alerts, etc.).
  4. Verify battery optimization settings if background monitoring is used.
  5. If needed: send a support email with logs (where available) and your exact steps to reproduce.
